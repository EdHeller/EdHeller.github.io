My name’s Ed, I’m 25 and I’m a computer programmer and web developer.
One of my favorite things in game development is to devise different solutions to the same problem. Generally there's a trade off in development time, Ram usage, Storage usage or CPU usage It's fun to balance these to meet the needs of different systems. It's always a learning process when creating different styles of game, code suited to one task might not be applicable to another. Modular and reusable code is great but it can cause trouble when the current system specifications change so often (The amazing code you made for 4way analog won't fair so well with a modern digital joystick). It tends to be the edage cases that get me I think other programmers can relate, Having a good set of test cases should mitigate most risks but taking time to QA can be just as important.
I was formerly studying game and interactive media at Tribeca Flashpoint Media Arts Academy of Chicago set to get my AS in 2014 before I took a leave of absence due to medical reasons. I learned a lot working directly with industry professionals and my peers and I feel I'm ready to take on any task that comes my way!
